# Future 接口简介

Future 接口实现了操作异步任务执行一些方法，如获取异步任务的执行结果，取消任务的执行，判断任务是否被取消，判断任务是否执行完毕等。

Future 接口可以为主线程开一个分支任务，专门为主线程处理耗时耗力的复杂业务。

Future 接口是 `Java5` 新加的一个<mark style="background: #FF5582A6;">接口</mark>，它提供了一种<mark style="background: #FFB86CA6;">异步并行</mark>计算的功能。

如果主线程需要执行一个很耗时的计算任务，我们就可以通过 future 把这个任务放到异步线程中执行。主线程继续处理其他任务或者先行结束，再通过 Future 获取计算结果。

# Runnable 接口和 Callable 接口

我们先来看下最简单的接口实现：

```java
class MyThread implements Runnable{

    @Override
    public void run() {

    }
}

class MyThread2 implements Callable<String> {

    @Override
    public String call() throws Exception {
        return null;
    }
}
```

可以看到，Runnable 接口和 Callable 接口的区别有：

- Runnable 接口实现的方法是 `run()`，Callable 接口是 `call()`
- Runnable 接口无返回，Callable 接口有返回值
- Runnable 接口不抛出异常，Callable 接口会抛出异常

## 目标

1. 多线程
2. 有返回值
3. 异步执行

可以看到
- Thread 类只接受 Runnable 接口, 因此没有返回值，和目标 2 冲突
- 不能异步执行

为了解决以上痛点，我们找到的 Runnable 接口的一个子接口：`RunnableFuture<V>`

```java
/**
 * A {@link Future} that is {@link Runnable}. Successful execution of
 * the {@code run} method causes completion of the {@code Future}
 * and allows access to its results.
 * @see FutureTask
 * @see Executor
 * @since 1.6
 * @author Doug Lea
 * @param <V> The result type returned by this Future's {@code get} method
 */
public interface RunnableFuture<V> extends Runnable, Future<V> {
    /**
     * Sets this Future to the result of its computation
     * unless it has been cancelled.
     */
    void run();
}
```

可以看到，`RunnableFuture` 接口继承了 Runnable 接口和 Future 接口。

有个重要的实现类： `FutureTask`

```java
/**
 * A cancellable asynchronous computation.  This class provides a base
 * implementation of {@link Future}, with methods to start and cancel
 * a computation, query to see if the computation is complete, and
 * retrieve the result of the computation.  The result can only be
 * retrieved when the computation has completed; the {@code get}
 * methods will block if the computation has not yet completed.  Once
 * the computation has completed, the computation cannot be restarted
 * or cancelled (unless the computation is invoked using
 * {@link #runAndReset}).
 *
 * <p>A {@code FutureTask} can be used to wrap a {@link Callable} or
 * {@link Runnable} object.  Because {@code FutureTask} implements
 * {@code Runnable}, a {@code FutureTask} can be submitted to an
 * {@link Executor} for execution.
 *
 * <p>In addition to serving as a standalone class, this class provides
 * {@code protected} functionality that may be useful when creating
 * customized task classes.
 *
 * @since 1.5
 * @author Doug Lea
 * @param <V> The result type returned by this FutureTask's {@code get} methods
 */
public class FutureTask<V> implements RunnableFuture<V> {
    /*
     * Revision notes: This differs from previous versions of this
     * class that relied on AbstractQueuedSynchronizer, mainly to
     * avoid surprising users about retaining interrupt status during
     * cancellation races. Sync control in the current design relies
     * on a "state" field updated via CAS to track completion, along
     * with a simple Treiber stack to hold waiting threads.
     */
//...
}
```

但是看到这里可能会有疑惑，`FutureTask` 类实现了 RunnableFuture 接口，那么只实现了 Runnable 接口和 Future 接口，但是没有和 Callable 接口扯上关系，也就是说还是获取不到返回值。

为了解决这个问题， FutureTask 类<mark style="background: #FF5582A6;">没有空参构造</mark>方法，必须要注入一个 Callable 接口：

```java
    /**
     * Creates a {@code FutureTask} that will, upon running, execute the
     * given {@code Callable}.
     *
     * @param  callable the callable task
     * @throws NullPointerException if the callable is null
     */
    public FutureTask(Callable<V> callable) {
        if (callable == null)
            throw new NullPointerException();
        this.callable = callable;
        this.state = NEW;       // ensure visibility of callable
    }
```

可以看到，通过构造方法可以注入一个 Callable 接口的实现类，从而满足我们上面的 3 个目标。

# FutureTask 结合线程池提升性能

```java
    // 三个任务，目前只有一个线程来处理，要耗时多久
    // 每个线程都需要执行500毫秒，总耗时大概1500毫秒
    private static void m1() {
        long startTime = System.currentTimeMillis();

        // 暂停毫秒
        try {
            TimeUnit.MILLISECONDS.sleep(500);
            TimeUnit.MILLISECONDS.sleep(500);
            TimeUnit.MILLISECONDS.sleep(500);
        } catch (Exception e) {
            e.printStackTrace();
        }

        long endTime = System.currentTimeMillis();
        System.out.println("---cost time: " + (endTime - startTime) + " 毫秒");
    }

    // 三个任务，使用线程池来执行，需要耗时多久
    // 三个线程一起执行，总耗时大概500毫秒
    private static void m2() throws InterruptedException, ExecutionException {
        long startTime = System.currentTimeMillis();

        ExecutorService threadPool = Executors.newFixedThreadPool(3);

        FutureTask<String> futureTask1 = new FutureTask<>(() -> {
            try {
                TimeUnit.MILLISECONDS.sleep(500);
            } catch (Exception e) {
                e.printStackTrace();
            }
            return "task1 over";
        });
        threadPool.submit(futureTask1);

        FutureTask<String> futureTask2 = new FutureTask<>(() -> {
            try {
                TimeUnit.MILLISECONDS.sleep(500);
            } catch (Exception e) {
                e.printStackTrace();
            }
            return "task2 over";
        });
        threadPool.submit(futureTask2);

        FutureTask<String> futureTask3 = new FutureTask<>(() -> {
            try {
                TimeUnit.MILLISECONDS.sleep(500);
            } catch (Exception e) {
                e.printStackTrace();
            }
            return "task3 over";
        });
        threadPool.submit(futureTask3);

        System.out.println(futureTask1.get());
        System.out.println(futureTask2.get());
        System.out.println(futureTask3.get());

        threadPool.shutdown();
        long endTime = System.currentTimeMillis();
        System.out.println("---cost time: " + (endTime - startTime) + " 毫秒");
    }
```

给出了两个方法，分别是顺序往下执行和使用线程池执行的方法，效果差距很明显：

# Future 的优缺点

## 优点

future+线程池异步多线程任务配合，能显著提高程序的执行效率

## 缺点

1. `get()` 方法会阻塞
2. `isDone()` 轮询的方法会耗费无畏的 CPU 资源，而且也不见得能及时得到计算结果。

案例：

```java
    // get()方法会导致阻塞
    private static void getTest() throws InterruptedException, ExecutionException, TimeoutException {
        FutureTask<String> futureTask = new FutureTask<>(() -> {
            try {
                System.out.println(Thread.currentThread().getName() + "\t -----come in");
                TimeUnit.SECONDS.sleep(5);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return "task over";
        });

        new Thread(futureTask, "t1").start();

        System.out.println(Thread.currentThread().getName() + "\t + 忙其他任务了");

        System.out.println(futureTask.get());

    }
```

执行结果：

<img src="https://coachhe-1305181419.cos.ap-guangzhou.myqcloud.com/Redis/20230508014228.png" width=50%>

可以看到，这个结果是符合预期的，主线程会先忙其他任务，然后获取 futureTask 的执行结果。

但是如果我们将打印 main 线程的任务往后移，就可能会导致因为 `get()` 方法阻塞而很迟才能打印 main 线程的情况：

```java
    // get()方法会导致阻塞
    private static void getTest() throws InterruptedException, ExecutionException, TimeoutException {
        FutureTask<String> futureTask = new FutureTask<>(() -> {
            try {
                System.out.println(Thread.currentThread().getName() + "\t -----come in");
                TimeUnit.SECONDS.sleep(5);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return "task over";
        });

        new Thread(futureTask, "t1").start();


        System.out.println(futureTask.get());
//        System.out.println(futureTask.get(3, TimeUnit.SECONDS));
        
        System.out.println(Thread.currentThread().getName() + "\t + 忙其他任务了");

    }
```

打印结果为：

<img src="https://coachhe-1305181419.cos.ap-guangzhou.myqcloud.com/Redis/20230508014454.png" width=50%>

程序会阻塞等待 get 方法返回之后，再继续执行主线程，这样其实是不符合预期的。

解决的方式有几种：

1. 使用 get 方法时添加超时参数：`utureTask.get(3, TimeUnit.SECONDS)`，这样超时之后会抛出异常，然后使用程序对异常进行处理
2. 使用 `isDone()` 方法轮询

但是这两种方法都不太优雅，是否有方式可以在程序执行完了之后主动通知我们，而不需要我们一直去查询执行中的程序呢？

# Future 接口改进

1. 对于简单的业务场景来说，使用 Future 完全没问题
2. 多个任务前后依赖可以组合处理的场景，Future 不太适合
3. 对多个任务比较计算速度的场景，Future 不太适合

阻塞方式和异步编程的设计理念相违背，而轮询的方式会耗费无谓的 CPU 资源。因此，JDK 8 设计出 CompletableFuture。

CompletableFuture 提供了一种<mark style="background: #FF5582A6;">观察者模式类似</mark>的机制。可以让任务执行完后通知监听的一方。

# CompletableFuture 类

```java
public class CompletableFuture<T> implements Future<T>, CompletionStage<T> {

}
```

可以看到，CompletableFuture 类实现了两个接口，一个 Future 接口，一个 CompletionStage 接口

## CompletionStage 接口

这个接口方法非常多：

<img src="https://coachhe-1305181419.cos.ap-guangzhou.myqcloud.com/Redis/20230509005509.png" width=50%>

- CompletionStage 接口代表异步计算过程中的某一个阶段，一个阶段完成以后可能会触发另外一个阶段
- 一个阶段的计算执行可以是一个 Function，Consumer 或者 Runnable。比如：`stage.thenApply(x -> square(x)).thenAccept(x -> System.out.print(x)).thenRun(() -> System.out.println())`
- 一个阶段的执行可能是被单个阶段的完成触发，也可能是由多个阶段一起触发

## CompletableFuture

- 在 Java 8 中，CompletableFuture 提供了非常强大的 Future 的拓展功能，可以帮助我们简化异步编程的复杂性，并且提供了函数式编程的能力，可以通过回调的方式处理计算结果，也提供了转换和组合 CompletableFuture 的方法
- 它可能代表一个明确完成的 Future，也有可能代表一个完成阶段（CompletionStage），它支持在计算完成以后触发一些函数或执行某些动作
- 它实现了 Future 和 CompletionStage 接口


## 核心的四个静态方法，来创建一个异步任务

### 静态方法创建的原因

直接 new 一个 CompletableFuture，出现的是一个不完美的 CompletableFuture。

### RunAsync, 无返回值

`RunAsync` 可以创建一个不带返回值的 CompletableFuture

#### 无线程池

来看下不带线程池的 RunAsync 静态方法：

```java
    /**
     * Returns a new CompletableFuture that is asynchronously completed
     * by a task running in the {@link ForkJoinPool#commonPool()} after
     * it runs the given action.
     *
     * @param runnable the action to run before completing the
     * returned CompletableFuture
     * @return the new CompletableFuture
     */
    public static CompletableFuture<Void> runAsync(Runnable runnable) {
        return asyncRunStage(ASYNC_POOL, runnable);
    }
```

可以看到，传入一个 Runnable 类型的参数，并且类型是 Void，代表无返回值，那么我们来实际使用一下：

```java

                TimeUnit.SECONDS.sleep(1);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        System.out.println(voidCompletableFuture.get());

}
```

可以看到，我们创建了一个 CompletableFuture 类，使用 lambda 表达式传入了一个 Runnable 类型接口。看下结果：

#### 有线程池

来看下带线程池的静态方法：

```java
    /**
     * Returns a new CompletableFuture that is asynchronously completed
     * by a task running in the given executor after it runs the given
     * action.
     *
     * @param runnable the action to run before completing the
     * returned CompletableFuture
     * @param executor the executor to use for asynchronous execution
     * @return the new CompletableFuture
     */
    public static CompletableFuture<Void> runAsync(Runnable runnable,
                                                   Executor executor) {
        return asyncRunStage(screenExecutor(executor), runnable);
    }
```

第二个参数就是传入的线程池。

### SupplyAsync 有返回值

#### 无线程池版本

先看下静态方法：

```java
    /**
     * Returns a new CompletableFuture that is asynchronously completed
     * by a task running in the {@link ForkJoinPool#commonPool()} with

     *
     * @param supplier a function returning the value to be used
```

可以看到，传入的是一个 `Supplier<U>` 类型的参数，

# CompletableFuture 基本使用

## 能完全完成 Future 的功能

```java
    public static void main(String[] args) throws ExecutionException, InterruptedException {

        ExecutorService executorService = Executors.newFixedThreadPool(3);

        CompletableFuture<Integer> integerCompletableFuture = CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + "---come in");
            int result = ThreadLocalRandom.current().nextInt(10); //产生一个随机数
            try {
                TimeUnit.SECONDS.sleep(1);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("1秒后出结果" + result);
            return result;
        }, executorService);

        System.out.println(Thread.currentThread().getName() + " 线程先去忙其他事了");
        System.out.println(integerCompletableFuture.get());

    }
```

可以看到，Future 接口能完成的功能，CompletableFuture 是可以完美完成的。

但是 Future 接口之前的两大问题：
1. `get()` 方法阻塞
2. `isDone()` 方法一直询问太耗费 CPU
在 CompletableFuture 这里也完美解决了。解决方案是将回调函数传入 `whenComplete()` 方法中，当 CompletableFuture 接口任务执行完成之后，会自动调用该回调函数。

## 减少阻塞和轮询 whenComplete

```java
    public static void main(String[] args) throws ExecutionException, InterruptedException {

        ExecutorService executorService = Executors.newFixedThreadPool(3);
        CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + " -----come in");
            int result = ThreadLocalRandom.current().nextInt(10);
            try {
                TimeUnit.SECONDS.sleep(1);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return result;
        }, executorService).whenComplete((v, e) -> { // 代表没有异常的情况
            if (e == null) {
                System.out.printf("=== 计算完成, 结果为%s ===%n", v);
            }
        }).exceptionally(e -> { // 有异常的情况
            e.printStackTrace();
            System.out.println("异常情况" + e.getCause() + ":\t" + e.getMessage());
            return null;
        });

        System.out.println(Thread.currentThread().getName() + "线程先去忙其他事情了");
        executorService.shutdown();

    }
    
```

一共使用了两个方法：

- `whenComplete()`，传入一个回调函数，其实就是一个 lambda 表达式，当执行完成之后，如果没有异常，就会调用该函数进行输出
- `exceptionally()`，当产生异常时，调用里面的方法进行处理

# 函数式接口总结

| 函数式接口名称 | 方法名称   | 参数     | 返回值   |
| -------------- | ---------- | -------- | -------- |
| Runnable       | `run()`    | 无参数   | 无返回值 |
| Function       | `apply()`  | 1 个参数 | 有返回值 |
| Consumer       | `accept()` | 1 个参数 | 无返回值 |
| Supplier       | `get()`    | 没有参数 | 有返回值 |
| BiConsumer     | `accept()` | 2 个参数 | 无返回值 | 

# 链式语法

传统语法中，如果需要使用 `set()` 方法给属性赋值，那么有多少个属性就需要使用多少次 `set()` 方法，比较繁琐：

```java
@Data
@ToString
@NoArgsConstructor
@AllArgsConstructor
@Accessors(chain = true)
class Student {
    private Integer id;
    private String studentName;
    private String major;
}
```

比如我这里要 set 多个属性，那么传统方式，我们就需要使用以下方法：

```java
public class CompletableFutureMallDemo {
    public static void main(String[] args) {
        Student student = new Student();
        student.setId(12);
        student.setStudentName("li4");
        student.setMajor("english");
    }
}
```

那么加上 `@Accessors(chain = true)` 之后，就可以使用以下方式：

```java
    public static void main(String[] args) {
        Student student = new Student();
        student.setId(12).setStudentName("li4").setMajor("english");
    }
```
