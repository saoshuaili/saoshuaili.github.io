# 乐观锁和悲观锁

## 悲观锁

-<mark style="background: #FF5582A6;"> 悲观锁</mark>认为自己在使用数据的时候一定有别的线程来修改数据，因此在获取数据的时候会先加锁，确保数据不会被别的线程修改。
- 悲观锁的<mark style="background: #FFF3A3A6;">实现方式</mark>
	- Synchronized 关键字
	- Lock 的实现类都是悲观锁
- 适合写操作多的场景，先加锁可以保证写操作时数据正确。显示的锁定之后再操作同步资源。
